Python Virtual Environments

A Python virtual environment is a container where you can save the necessary packages used by your script programs.

Advantages

  • Install packages without superuser privileges.

  • Don't worry about mess conflicts with actual version in your system.

  • Don't pollutes your list of modules used in your project.

Disadvantages

  • You need create for each project a virtual environment

  • This virtual environments creates copies of your binary python interpreter and package manager, generating more bytes for your computer, increasing the used of your disk.

Standard Virtual Environments

Python has in your source code a standard virtual environment called virtualenv, with a great command line venv.

virtualenv PEP 405

Create Environment


python -m venv /home/user/.venv

Activate Environment

To used the copy of the Python binaries you need to activate this environment.


source /home/user/.venv/bin/activate

Packages Management

Generate packages list


pip freeze

Generate packages list to a file 


pip freeze > requirements.txt

Install list of packages from a file 


pip install -r requirements.txt
